# CS230-Operating-Platforms-Portfolio

## **The Gaming Room  Software Design Template**  
The client, **The Gaming Room**, wanted to expand their existing game *Draw It or Lose It* into a **web‑based, multi‑platform application**. Their goal was to support multiple teams, multiple players, and ensure that game instances were unique and properly managed. They needed a design that would allow the game to run on various operating platforms while maintaining consistency, security, and scalability.

## **What I Did Well in the Documentation**  
I did particularly well in clearly outlining the **architecture and design decisions** for the application. My documentation explained how the system should manage game instances, teams, and players using object‑oriented principles. I also provided strong justification for using the **Singleton pattern** to ensure only one instance of the game service exists, which prevents duplicate games and maintains data integrity.

## **Helpful Parts of the Design Process**  
Working through the design document helped me understand how important it is to plan before coding. Breaking down the system into classes, responsibilities, and interactions made the development process much easier. Creating UML diagrams and identifying design patterns gave me a clear roadmap, reducing confusion once I started writing code.

## **What I Would Revise and Improve**  
If I could revise one part of my documentation, I would improve the **UML diagrams** by adding more detail and ensuring all relationships and methods were fully represented. I would also refine some of the explanations to make them more concise and easier for a development team to follow.

## **Interpreting User Needs and Implementing Them**  
I interpreted the user’s needs by focusing on what the client wanted the game to do: support multiple players, prevent duplicate game instances, and run smoothly across platforms. I implemented these needs by designing classes that manage players, teams, and game instances in a structured way. Considering user needs is essential because it ensures the final product actually solves the client’s problem and provides a good user experience.

## **My Approach to Designing Software**  
I approached the design using **object‑oriented principles**, UML modeling, and design patterns. I analyzed the requirements, broke the system into logical components, and documented how each part should behave. In the future, I would continue using techniques like UML diagrams, design patterns, and requirement analysis because they help create clear, maintainable, and scalable software designs.
